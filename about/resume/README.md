## 项目经历(浪潮通信信息)
### 移动综合资源管理系统
- 工作经历：在浪潮的这段时间里，我主要负责黑龙江研发团队的需求对接及确认排期，研发任务派发，测试框架问题后与山东本部产品线对接。
本人的研发工作范围包含：移动综合资源管理系统的专业流程的大量二开工作以及部分的产品自研框架的开发工作、本地二开流程的前后台的框架搭建。
技术范围包含了：前后台的代码开发、服务器运维、基于框架的配置实施、数据的迁移汇总、部分的需求整理、需求确认、底层数据模型搭建。
工作期间亮点：当时所处环境下，承接大量前离职人员开发了一半且BUG较多的流程项目，陆续的在将近一年的时间中，在甲方客户对我方公司极不满意的情况下，
组织研发团队重构了当时业务较为简单的流程项目（反复测试几乎没有BUG后上线），在这个之后又重写了流程二开的代码架构，建立了一些规范与约定。
较为复杂的且已经上线的流程项目由本人承担，由于有着大量的存量工单问题以及代码质量的惨不忍睹且没有需求文档及业务文档的情况之下，协调询问各方人员将主要业务梳理清晰后，
阅读代码细节寻找业务😂。终于在将近3个月的时间中，大量的删除无用冗余的代码，修改大量存在高危BUG的代码逻辑，优化前台的AJAX异步请求及DOM树结构，优化后台代码结构及算法，
最后流程趋于稳定，大大提升且保护了数据质量。此后又在此基础上了研发了大量业务，此流程项目乃是浪潮在全国范围内最复杂的流程项目，在我接手此项目后从未有过重大故障，
获得了移动甲方客户及众多对接三方客户的信任与高度评价。哦，对了，该流程项目的名字叫：基站入网。流程图请参考下面的图片：**流程配置-流程图**
